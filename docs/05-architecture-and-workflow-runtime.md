# Architecture and Workflow Runtime

## When a State Machine Is Not Enough

A state machine describes allowed transitions. A workflow runtime must also handle:

- Scheduling
- Worker leases
- Retries
- Timers
- Idempotency
- Crash recovery
- External provider calls
- Stream events
- Materialized read models
- Audit and replay

Treat long-running product workflows as a small durable execution system.

## Minimal Durable Runtime Shape

Recommended layers:

- `operation_runs`: user or agent intent.
- `agent_actions`: planned user-facing actions and approval state.
- `workflow_runs`: execution-level workflow instance.
- `workflow_events`: append-only event log.
- `workflow_commands`: typed command/outbox rows generated by reducers.
- `workflow_activity_runs`: bounded side effects.
- `workflow_activity_attempts`: retries, remote refs, errors, durations.
- `workflow_entity_deltas`: why an entity was added, ignored, quarantined, or projected.
- Current-state tables: materialized operational state.
- Read models: optimized public API projections.

## Reducer Rule

Workers should write results and events. Reducers or command owners should generate downstream commands.

Do not allow arbitrary workers to start the next stage directly.

## Activity Rule

Every bounded side effect should be an activity:

- Provider submit
- Poll/fetch
- File/object write
- Model adjudication
- Export generation
- Projection build
- Notification

Activities need idempotency keys, owner, retry policy, timeout, and causal links to commands/events.

## Read Model Rule

Read models serve users. They should not:

- Repair workflow state.
- Stitch fallback data from legacy paths.
- Trigger provider calls.
- Infer missing lifecycle state.

If data is not ready, fail closed or return explicit readiness.

## Agent-Callable Rule

Agents should be able to:

- Query state and provenance.
- Request retry/cancel/resume through operations.
- Start bounded commands through action registry.
- See approval requirements.
- Inspect why entities did or did not enter projections.

Agents should not bypass owners.
